# Implementation Plan - Release 2.1.0 Fixes

## Problem Analysis

1. **Character Inconsistency**: The system ignores detailed character descriptions. `webtoon.py` uses a placeholder generic string (`"{name}: {gender} character (reference image provided...)"`) instead of the rich `visual_description` generated by the Story Analyst. This renders the style sheet ineffective.
2. **Disconnected Workflow**: The `visual_prompter` agent generates "Enhanced Prompts" but these are never saved back to the `WebtoonPanel` objects. The final image generation uses the raw, unpolished prompt from the initial draft.
3. **Prompt Redundancy (Bloat)**: The `visual_prompter` generates a full prompt (Character + Visual + Style + Tech) which is then wrapped inside `SCENE_IMAGE_TEMPLATE` (which has its own Character + Tech slots). This creates massive, redundant prompts that confuse the model and waste tokens ("Pruning" required).
4. **Environment Loss**: Truncation logic was too aggressive (fixed in previous step, but needs verification with correct workflow connection).

## Proposed Changes

### 1. Fix Character Descriptions in `app/routers/webtoon.py`

**Goal**: Ensure the rich character descriptions are actually passed to the image generator template.

- Update `generate_scene_image` to use `char.get('visual_description')` instead of the generic placeholder string.
- This ensures the `{character_description}` slot in `SCENE_IMAGE_TEMPLATE` is populated with the detailed consistency data.

### 2. Prune `visual_prompter.py` for Template Integration

**Goal**: Remove redundant blocks that `SCENE_IMAGE_TEMPLATE` already handles.

- Modify `visual_prompter.py` to **SKIP** generating `[CHARACTER CONSISTENCY]` and `[TECHNICAL]` blocks by default.
- The `visual_prompter` should focus on enhancing the **Scene Description** (`[VISUAL]`) and **Style/Mood** (`[STYLE & MOOD]`).
- This makes the enhanced prompt fit perfectly into the `{visual_prompt}` slot of the template without duplicating instructions.

### 3. Connect Workflow in `app/workflows/enhanced_webtoon_workflow.py`

**Goal**: Ensure the enhanced prompts are used.

- Update `visual_prompter_node` to write the `enhanced_prompts` back into the `state["panels"]` (specifically updating the `visual_prompt` field of each panel).
- This ensures the `webtoon_writer`/`panel_composer` results contain the optimized prompts when sent to the frontend/API.

## Verification

- **Test**: Generate a scene image (using existing tests or manual trigger).
- **Check**: Inspect logs to see:
  1. `final_prompt` contains rich character descriptions at the top (from Template).
  2. `final_prompt`'s `<master_visual_description>` contains the enhanced visual/style/mood info.
  3. No duplicate `[CHARACTER]` or `[TECHNICAL]` blocks.
